Test Progetto-C
=======
Traccia 2: Sistema di Gestione degli Accessi e delle Prenotazioni di
    un’Aula Studio (Gruppo di 2 persone)
    Obiettivo:
    Sviluppare un programma in C che gestisca l’utilizzo di un’aula studio universitaria. Il
    sistema deve permettere di registrare gli studenti, gestire le prenotazioni dei posti,
    controllare gli accessi effettivi, monitorare le presenze e produrre report
    sull’occupazione dell’aula.

Requisiti:
    ● Registrazione di uno studente con:
        ○ matricola
        ○ nome
        ○ corso di laurea
    ●Inserimento di una prenotazione con:
        ○ matricola dello studente
        ○ data
        ○ fascia oraria
        ○ posto assegnato
    ● Verifica della disponibilità dei posti per fascia oraria
    ● Gestione del check-in di uno studente prenotato
    ● Gestione dell’ingresso senza prenotazione (se disponibili posti liberi)
    ● Registrazione dell’uscita di uno studente
    ● Visualizzazione degli studenti:
        ○ prenotati
        ○ presenti
        ○ in attesa
    ● Gestione di una lista di attesa
    ● Annullamento di una prenotazione
    ● Aggiornamento automatico dei posti disponibili
    ● Visualizzazione dello storico degli accessi
    ● Generazione di un report con:
        ○ numero totale di prenotazioni
        ○ accessi effettivi
        ○ occupazione per fascia oraria
        ○ studenti che non si presentano
        ○ studenti rimasti in attesa
Casi di Test:
    ● Verifica della registrazione degli studenti
    ● Test dell’inserimento delle prenotazioni
    ● Verifica della disponibilità dei posti
    ● Test del check-in e check-out
    ● Verifica dell’ingresso senza prenotazione
    ● Test della lista di attesa
    ● Verifica dell’annullamento e aggiornamento disponibilità
    ● Test dello storico e dei report
Report:
    ● Motivazione della Scelta dell'ADT: Illustrare perché specifici ADT sono stati
    scelti per gestire studenti, prenotazioni, attese, presenze e ricerche.
    ● Progettazione: Illustrare la struttura del sistema e le interazioni tra i
    componenti.
    ● Specifica Sintattica e Semantica: Descrivere operazioni, pre/post condizioni,
    input/output ed effetti collaterali.
    ● Razionale dei Casi di Test
